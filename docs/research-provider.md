# 现实资料检索

解读在服务端依次执行意图判断、查询规划、受限搜索、URL 校验/去重/截断，再把资料包交给 AI。网页标题、摘要、URL 与用户问题均作为不可信数据处理，不能改变系统规则。

## 配置 Tavily

在部署环境（不要提交密钥）设置：

```env
RESEARCH_PROVIDER=tavily
RESEARCH_API_KEY=...
RESEARCH_TIMEOUT=12000
RESEARCH_MAX_SOURCES=6
```

当前实现选择 Tavily Search API：它是显式的搜索 API，而非假设聊天模型拥有浏览器能力。请求只从 Route Handler 触发的服务端编排中发出；API Key 不会发送到浏览器。每个请求有 3–30 秒超时、最多一次重试、最多 8 个来源，失败会记录脱敏错误类别并降级到普通塔罗解读。

`mock` 是默认值：不进行网络请求、不生成虚构来源；对明显需要事实的问题，结果页显示“当前为离线演示模式，未连接实时资料服务”。非事实型自我反思问题会跳过检索。

请根据 Tavily 当前套餐确认费用、速率限制及数据处理条款。不要把敏感个人信息放进问题；问题、查询和网页摘要都有限长，日志会脱敏问题与密钥。检索结果只是可点击的线索，关键结论应由用户打开原始来源复核。
